About

I'm a software engineer focused on developer tooling and AI engineering — building the systems that make complex software legible and controllable. The through-line of my work is closing the loop between what a system is doing and what you can observe and change about it: traffic inspection, runtime tooling, and agent-ready integrations.

I lead development of Busymate DevTools, an AI-first platform for capturing, inspecting, mocking, and replaying HTTPS traffic across iPhones, Android, browsers, and device farms — unified in one realtime dashboard, with an AI teammate (BusyBro), a large MCP tool surface, REST/WebSocket APIs, and enterprise-grade security.

Earlier work spans mobile tooling (Capacitor native plugins and SwiftUI components) and web-runtime tooling (debuno — bringing Deno-style module resolution to Bun and Node). I organize this work through the BusyBrothers team and have been publishing open-source software on GitHub since 2011.

Busymate DevTools busymate.dev ↗

AI-first HTTPS traffic capture, inspection, mocking, and replay for development teams debugging across every device — unified in one realtime dashboard.

  • ▸Capture everywhere — on-device VPN for iOS, MITM proxy for Android & browsers, zero-install CLI for Chrome, and device farms at scale.
  • ▸Inspect & manipulate — realtime dashboard with filtering, breakpoints, hand-editing, and scripted mocking via a sandboxed JS engine.
  • ▸AI teammate (BusyBro) — persistent memory, integrated directly into debugging workflows.
  • ▸Agent-ready — a 303-tool MCP server for Claude, Cursor, and other agents; REST & WebSocket APIs for programmatic access.
  • ▸Enterprise security — database-enforced RBAC, audit trails, and an encrypted vault.

Earlier devtooling: debuno (Deno module resolution for Bun & Node), Capacitor native plugins for iCloud, and SwiftUI components. Mobile-first, runtime-curious.
